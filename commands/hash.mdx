sttr provides various hashing algorithms for checksums, file verification, and password hashing.

## Cryptographic Hashes

### MD5

<Card title="md5" icon="fingerprint">
  Generate MD5 hash (128-bit)
</Card>

```bash
# Generate MD5 hash
echo -n "password" | sttr md5
# Output: 5f4dcc3b5aa765d61d8327deb882cf99

# Hash a file
cat file.txt | sttr md5

# Hash without newline
echo -n "text" | sttr md5
```

**Aliases:** `md5-sum`

**Note:** MD5 is not recommended for security-critical applications. Use SHA-256 or higher for better security.

## Password Hashing

### bcrypt

<Card title="bcrypt" icon="key">
  Generate bcrypt hash (password hashing)
</Card>

```bash
# Generate bcrypt hash (default 10 rounds)
echo -n "password" | sttr bcrypt
# Output: $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy

# Increase security with more rounds
echo -n "password" | sttr bcrypt -r 12
# More computation = more secure

# Production password hashing
echo -n "user_password" | sttr bcrypt -r 14
```

**Aliases:** `bcrypt-hash`

**Flags:**
- `-r, --number-of-rounds` - Computational cost (default: 10, recommended: 10-14)

**Note:** bcrypt is designed for password hashing. Higher rounds = more secure but slower. Each increment doubles the time. Recommended: 12-14 for production.

## BLAKE2 Hashing

### BLAKE2b

<Card title="blake2b" icon="bolt">
  Generate BLAKE2b hash (faster than SHA-2)
</Card>

```bash
# Generate BLAKE2b hash (64 bytes default)
echo -n "data" | sttr blake2b
# Output: 508c5e8c327c14e2e1a72ba34eeb452f37458b209ed63a294d999b4c86675982...

# Custom hash size (1-64 bytes)
echo -n "data" | sttr blake2b -s 32
# Output: 32-byte hash

# Minimum size
echo -n "data" | sttr blake2b -s 1
```

**Aliases:** `blake2b-hash`, `blake2b-sum`

**Flags:**
- `-s, --size` - Hash size in bytes (1-64, default: 64)

**Use cases:** High-performance hashing, file integrity, checksums

---

## Checksums

### CRC32

<Card title="crc32" icon="check-double">
  Generate CRC32 checksum
</Card>

```bash
# Generate CRC32 checksum (IEEE polynomial)
echo -n "data" | sttr crc32
# Output: 12345678

# Use different polynomial
echo -n "data" | sttr crc32 -p castagnoli
echo -n "data" | sttr crc32 -p koopman

# Quick file verification
cat package.zip | sttr crc32
```

**Aliases:** `crc32-sum`, `crc32-checksum`

**Flags:**
- `-p, --polynomial` - CRC32 polynomial (ieee, castagnoli, koopman)
  - `ieee` - IEEE 802.3 (default, most common)
  - `castagnoli` - Castagnoli (CRC32C)
  - `koopman` - Koopman

**Use cases:** Error detection, file integrity, ZIP/PNG formats

## Comparison Guide

### For Security (Cryptographic)

<Tabs>
  <Tab title="Password Hashing">
    **Use:** `bcrypt`
    
    Specifically designed for passwords with adjustable computational cost.
    
    ```bash
    echo -n "password" | sttr bcrypt -r 12
    ```
  </Tab>
  
  <Tab title="General Purpose">
    **Use:** `sha256` or `sha512`
    
    Industry standard for secure hashing.
    
    ```bash
    echo -n "data" | sttr sha256
    ```
  </Tab>
  
  <Tab title="High Performance">
    **Use:** `blake2b` or `blake2s`
    
    Faster than SHA-2, still cryptographically secure.
    
    ```bash
    echo -n "data" | sttr blake2b
    ```
  </Tab>
</Tabs>

### For Performance (Non-Cryptographic)

<Tabs>
  <Tab title="Maximum Speed">
    **Use:** `xxh-64` or `xxh-128`
    
    Extremely fast, good for hash tables and quick checksums.
    
    ```bash
    cat large_file | sttr xxh-64
    ```
  </Tab>
  
  <Tab title="Error Detection">
    **Use:** `crc32`
    
    Standard for error detection in networking and file formats.
    
    ```bash
    cat file.zip | sttr crc32
    ```
  </Tab>
</Tabs>

## Use Cases

### File Integrity Verification

```bash
# Download and verify
curl -O https://example.com/file.iso
cat file.iso | sttr sha256
# Compare with published SHA-256 hash

# Quick local verification
cat backup.tar.gz | sttr xxh-64 > backup.xxh64
# Later: verify
cat backup.tar.gz | sttr xxh-64
cat backup.xxh64
```

### Password Storage

```bash
# Hash password for storage (never store plain text!)
read -s PASSWORD
echo -n "$PASSWORD" | sttr bcrypt -r 12
# Store the output hash in database
```

### Data Deduplication

```bash
# Generate content hash
cat document.pdf | sttr sha256
# Use hash as filename or identifier
```

### API Signatures

```bash
# Create HMAC-style signature
echo -n "timestamp=123&data=value" | sttr sha256
```
